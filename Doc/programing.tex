\chapter{プログラミング}

\section{考え方}

MONTSUQIのアプリケーションを構成する基本要素は、LD(Logical Destination 
論理宛先)である。これはワークフローコントローラが処理するメッセージが
表現するものである。

LDは複数のウィンドウによって構成される。プレゼンテーションサーバはクライ
アントからのイベントがどのウィンドウに含まれるかを調べ、ウィンドウが含ま
れるLDにメッセージを送る。

MONTSUQIのアプリケーションは、イベントによって駆動される。このイベントは、

\begin{itemize}
\item	イベントの発生したウィンドウ名
\item	イベントの発生したウィジェット名
\item	イベントの内容を表現する文字列
\item	イベントの種類
\item	イベント発生時の画面データ
\end{itemize}

によって構成される。このイベントを使い処理を行う。

\section{定義体}

MONTSUQIを動かすためには、以下に示す定義体を用意する必要がある。

\begin{itemize}
\item	{\bf LD定義体}
\item	{\bf 画面レコード定義体}
\item	{\bf SPAレコード定義体}
\end{itemize}

が必要となる。また、画面(ウィンドウ)を表示するためには、画面定義体が必要
となる。
